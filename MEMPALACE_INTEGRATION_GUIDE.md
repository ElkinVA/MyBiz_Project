# 📜 Инструкция для следующего чата: Интеграция MemPalace

## ⚠️ Критически важная информация о среде

### Текущее состояние ресурсов:
- **Доступное место на диске**: ~16 МБ (всего 504 МБ, занято 97%)
- **Оперативная память**: ~678 МБ свободно из 1058 МБ
- **Увеличить ресурсы нельзя** — это фиксированные лимиты облачной песочницы

### Проблема предыдущей интеграции:
Попытка установить MemPalace с ChromaDB провалилась из-за нехватки места. 
ChromaDB + зависимости занимают ~200-300 МБ, что превышает доступные 16 МБ.

---

## ✅ РЕШЕНИЕ: Легковесная альтернатива

**Не используйте оригинальный MemPalace с ChromaDB!** Вместо этого реализуйте встроенную систему памяти на базе существующих компонентов проекта:

### Архитектура легковесной памяти:
```
Django Project
├── PostgreSQL/SQLite (уже есть)
├── Django Models (уже есть)
└── Custom Memory App (новое, ~100 КБ)
    ├── models.py (Chat, Message, Context)
    ├── services.py (save_chat, restore_context, search)
    └── signals.py (авто-сохранение при завершении)
```

### Преимущества:
- ✅ Не требует дополнительных зависимостей
- ✅ Занимает ~100-200 КБ вместо 300 МБ
- ✅ Использует существующую БД проекта
- ✅ Работает в текущих лимитах ресурсов
- ✅ Полностью контролируется вами

## 📋 Пошаговый план для следующего чата

### Шаг 1: Очистка места (если нужно)
```bash
# Удалить временные файлы
rm -rf /workspace/staticfiles/*
rm -rf /workspace/media/*
# Очистить кэш Python
find /workspace -type d -name "__pycache__" -exec rm -rf {} +
find /workspace -type f -name "*.pyc" -delete
```

### Шаг 2: Создать приложение `chat_memory`
```bash
cd /workspace
python manage.py startapp chat_memory
```

### Шаг 3: Реализовать модели
Файл: `chat_memory/models.py`
- `ChatSession` (id, session_id, started_at, ended_at, metadata)
- `Message` (id, session, role, content, timestamp, embedding_summary)
- `ContextSnapshot` (id, session, summary, key_points, restored_at)

### Шаг 4: Реализовать сервисы
Файл: `chat_memory/services.py`
- `save_chat_session(session_id, messages)` — сохранение диалога
- `restore_context(session_id)` — восстановление контекста
- `search_memories(query, limit=5)` — поиск по истории
- `generate_summary(messages)` — краткое содержание (без ML, просто текст)

### Шаг 5: Настроить сигналы
Файл: `chat_memory/signals.py`
- Авто-сохранение при завершении сессии
- Логирование в файл `/workspace/chat_history.jsonl`

### Шаг 6: Интегрировать в Django
- Добавить `chat_memory` в `INSTALLED_APPS`
- Создать миграции: `python manage.py makemigrations chat_memory`
- Применить: `python manage.py migrate`

### Шаг 7: Создать API для работы с памятью
Файл: `chat_memory/views.py` или в существующем `api/`
- POST `/api/memory/save/` — сохранить текущий чат
- GET `/api/memory/restore/<session_id>/` — восстановить контекст
- GET `/api/memory/search/?q=query` — поиск

### Шаг 8: Настроить автоматическое сохранение
- При завершении чата вызывать `save_chat_session()`
- Сохранять в JSONL файл для резервного копирования
- Опционально: сжимать старые записи

### Шаг 9: Тестирование
```bash
pytest tests/test_chat_memory.py -v
```

### Шаг 10: Документация
Создать `CHAT_MEMORY_SETUP.md` с инструкцией для пользователя

## 🚫 Чего НЕ делать в следующем чате

1. **НЕ устанавливать** `mempalace`, `chromadb`, `sentence-transformers` — они займут всё место
2. **НЕ клонировать** внешние репозитории в `/workspace`
3. **НЕ создавать** виртуальные окружения внутри проекта (используйте системное)
4. **НЕ загружать** большие модели машинного обучения

---

## 💡 Альтернативный вариант: Внешний сервис

Если нужна полноценная система с векторным поиском:
1. Разверните MemPalace на отдельном сервере/VPS
2. Интегрируйтесь через REST API
3. В Django храните только ссылки и мета-данные

Это снимет нагрузку с `/workspace`, но потребует внешней инфраструктуры.

## 📊 Мониторинг ресурсов

Перед установкой любых зависимостей всегда проверяйте:
```bash
df -h /workspace  # место на диске
free -m           # оперативная память
du -sh /workspace/* | sort -hr  # самые большие папки
```

Если доступно < 50 МБ — сначала очистите место!
